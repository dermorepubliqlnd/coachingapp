ⓘ How to use this import

One template handles both new users and updates. The system matches each row to an existing user by Employee ID. Matched = update. Unmatched = create new account.

Col Column name Required? Notes
AemployeeIdAlwaysKey used to match existing users
BfirstNameNew usersFirst name
ClastName—Last name
DemailNew usersCannot be changed for existing users
Eposition—Job title / position
FhireDate—Format: YYYY-MM-DD. Compliance starts month after hire.
GroleCoach—true / false — assign Coach role
HroleCoachee—true / false — assign Coachee role
IroleDirector—true / false — assign Director role
JpermDashboard—true / false — grant Dashboard access (non-directors)
KpermSessionRecords—true / false — grant Session Records access (non-directors)
LcoachEmail—Email of assigned coach (for coachees)
Mactive—true / false — defaults to true if blank

Lines starting with # in the CSV are treated as comments and skipped. Directors always have full access — permDashboard and permSessionRecords only affect non-director users.
